🔧 기술 플로우 (백엔드에서 일어나는 일)

[Phase 1 제출]
  POST /api/proof/:id/phase-result
    body: { phase_no: 1, submitted_json: ["B","D","A","C"] }
  ↓
  서버: proof_phases.model_answer ("B,D,A,C")와 비교
    → 일치: verify_status='PASS', score=100
    → 부분: 정확히 맞은 카드 개수 / 전체
  ↓
  INSERT INTO proof_phase_results
  ↓
  응답: { status: 'PASS', correct_count: 4, total: 4, next_phase: 2 }

[Phase 2 제출]
  POST /api/proof/:id/phase-result
    body: { phase_no: 2, submitted_json: {"blank_1":"평행","blank_2":"동위각",...} }
  ↓
  서버: proof_phase_items.correct_answer와 각 블랭크 비교
    → proof-verify.ts (TS 경량 verifier: 분수/수치 정규화 포함)
  ↓
  응답: { status: 'PARTIAL', correct: {"blank_1":true, "blank_2":false, ...}, hint: "..." }

[Phase 3 제출]
  POST /api/proof/:id/phase-result
    body: { phase_no: 3, submitted_json: { text: "..." } }
  ↓
  verify_status = 'PENDING_REVIEW'  (자동채점 없음)
  교사 대시보드 큐에 뜸
  ↓
  [교사가 승인]
  PATCH /api/proof-review/:result_id
    body: { status: 'PASS', comment: "..." }
  ↓
  서버: proof_results.box + 1 (최대 5)
        last_review = now
        next_review = now + interval(box)
  ↓
  푸시/앱내 알림 → 학생: "승인됐어요"

[세션 완료]

# E2E 테스트 가이드 - Admin Dashboard Live Tests

## 📋 테스트 개요

**목표:** Cloudflare Pages 라이브 서버에서 Admin Dashboard의 모든 기능을 검증

**테스트 파일:** `apps/desktop/e2e/admin-live.spec.ts`

**총 유즈케이스:** 10개 (UC-1 ~ UC-10)

---

## 🎯 유즈케이스 명세

### UC-1: 관리자 대시보드 접근 및 기본 렌더링
```
목적: 관리자 페이지 정상 로드 확인
단계:
  1. /admin 페이지 접속
  2. 페이지 제목 확인 (WAWA)
  3. 관리자 UI 요소 렌더링 확인
  4. 탭 네비게이션 (학생 관리, 시스템 설정) 확인

예상 결과: ✓ 모든 요소가 보임
```

### UC-2: 학생 관리 탭 - 학생 목록 및 필터링
```
목적: 학생 관리 탭의 기본 기능 확인
단계:
  1. "학생 관리" 탭 클릭
  2. 학생 목록 테이블 렌더링 확인
  3. 검색 입력창 확인
  4. 학년 필터 드롭다운 동작 확인
  5. 학생 추가 버튼 확인

예상 결과: ✓ 필터와 목록이 정상 작동
```

### UC-3: 학생 추가 모달 - 폼 열기 및 닫기
```
목적: 학생 추가 모달의 접근성과 폼 구조 확인
단계:
  1. 학생 관리 탭 열기
  2. 학생 추가 버튼 클릭
  3. 모달 팝업 확인
  4. 폼 필드 확인 (이름, 학년, 과목, 일정)
  5. 취소 버튼으로 모달 닫기

예상 결과: ✓ 모달이 정상적으로 열리고 닫힘
```

### UC-4: 시스템 설정 탭 - 설정 화면 로드
```
목적: 시스템 설정 페이지의 구조 확인
단계:
  1. "시스템 설정" 탭 클릭
  2. 설정 섹션 렌더링 확인
  3. Notion, AI, Kakao 설정 영역 확인

예상 결과: ✓ 설정 페이지가 정상 로드됨
```

### UC-5: 통계 정보 표시
```
목적: 학생 관리 페이지의 통계 정보 확인
단계:
  1. 학생 관리 탭 열기
  2. 총 학생 수, 활성/비활성 통계 확인

예상 결과: ✓ 통계가 표시됨 (데이터가 없어도 UI 존재)
```

### UC-6: 페이지네이션 UI 확인
```
목적: 많은 학생 데이터 처리 시 페이지네이션 작동 확인
단계:
  1. 학생 관리 탭 열기
  2. 페이지네이션 컨트롤 확인
  3. 이전/다음 버튼 또는 페이지 번호 클릭 가능 확인

예상 결과: ✓ 페이지네이션이 정상 작동 (또는 데이터 없으면 미표시)
```

### UC-7: 모바일 반응형 디자인
```
목적: 모바일 화면(375x667)에서 레이아웃 확인
단계:
  1. 뷰포트 크기를 375x667로 설정
  2. 학생 관리 탭 열기
  3. 필터 바와 콘텐츠 접근성 확인
  4. 터치 가능한 버튼 크기 확인

예상 결과: ✓ 모바일에서도 사용 가능함
```

### UC-8: 폼 유효성 검사
```
목적: 필수 필드 입력 검증 확인
단계:
  1. 학생 추가 모달 열기
  2. 이름 입력 없이 제출 시도
  3. 과목 선택 없이 제출 시도
  4. 에러 메시지 또는 유효성 검사 표시 확인

예상 결과: ✓ 에러 메시지나 제출 불가 상태 표시
```

### UC-9: 탭 전환 시 상태 유지 (선택사항)
```
목적: 탭 전환 후 이전 필터/선택이 유지되는지 확인
단계:
  1. 학생 관리 탭 열기
  2. 학년 필터 설정 (예: 중1)
  3. 시스템 설정 탭으로 전환
  4. 다시 학생 관리 탭으로 돌아오기
  5. 이전 필터 상태 확인

예상 결과: ✓ 필터가 유지되거나 초기화됨 (설계에 따라)
```

### UC-10: 성능 - 페이지 로드 시간
```
목적: 페이지 로드 성능 측정
단계:
  1. 페이지 로드 시간 측정
  2. 로드 완료까지 걸린 시간 기록

예상 결과: ✓ 10초 이내 로드됨
```

## 🚀 테스트 실행 방법

### 로컬 preview 서버에서 실행

```bash
cd apps/desktop

# 1. Preview 서버 실행 (백그라운드)
npm run preview &

# 2. E2E 테스트 실행
npm run test:e2e
```

### 라이브 Cloudflare Pages 서버에서 실행

```bash
# Cloudflare Pages URL로 테스트
E2E_BASE_URL=https://your-app.pages.dev npm run test:e2e
```

### 라이브 서버 + 특정 테스트만 실행

```bash
# 특정 유즈케이스만 실행
E2E_BASE_URL=https://your-app.pages.dev npm run test:e2e -- --grep "UC-1"

# 모바일 반응형만 테스트
E2E_BASE_URL=https://your-app.pages.dev npm run test:e2e -- --grep "UC-7"
```

## 🐛 문제 해결

### 테스트 타임아웃

```
TimeoutError: Page.goto timed out
→ 서버가 실행 중인지 확인
→ BASE_URL 확인
→ 포트 번호 확인 (preview는 4173, dev는 5173)
```

### 요소를 찾을 수 없음

```
Error: No element with text "..."
→ 로케이터 선택자 확인
→ 페이지가 완전히 로드될 때까지 대기
→ 콘솔 오류 확인
```

### API 호출 실패

```
Network error: 404
→ API 서버 실행 상태 확인
→ CORS 설정 확인
→ 로그에서 오류 메시지 확인
```
